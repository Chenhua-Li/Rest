# BPTT 与 REST-S 在线学习对比

这个文件夹提供一个最小可运行示例：`bptt_vs_rests_demo.py`。

运行方式：

```bash
python test/bptt_vs_rests_demo.py
```

示例任务是一个简单的时序二分类：输入是 `[time, batch, 2]` 的随机序列，如果第 0 个通道在整个时间窗内的累积和大于第 1 个通道，标签为 1，否则为 0。

## 两者相同点

- 使用同一个网络结构：`2 -> 16 -> 2`。
- 隐藏层都是 recurrent LIF 神经元。
- 输出层都是线性 readout。
- 初始参数相同，方便比较学习过程。

## BPTT 学习过程

BPTT 使用 `temporal_detach=False`。这意味着神经元状态会保留跨时间步的计算图。

一个 batch 的学习过程是：

1. 按时间步前向传播整个序列。
2. 把每个时间步的 loss 累加。
3. 在序列结束后，对完整时间计算图执行一次 `loss.backward()`。
4. 优化器更新一次参数。

直观上，BPTT 是“先看完整段时间，再沿时间反向传播误差”。

## REST-S 在线学习过程

REST-S 使用 `temporal_detach=True`，并通过 `get_rests(model, back="r")` 包装基础 SNN。

一个 batch 的学习过程是：

1. 每个时间步前向传播。
2. 前向时 REST-S 同步更新在线敏感度矩阵 `N`。
3. 当前时间步产生 `loss_t` 后，调用 `model.calc_grad(loss_t)`。
4. REST-S 用当前局部误差和 `N` 矩阵生成参数梯度。
5. 示例里每个时间步都执行一次 `optimizer.step()`。

直观上，REST-S 是“边看时间序列，边维护梯度所需的在线状态，并可边到达边更新参数”。

## 关键区别

| 维度 | BPTT | REST-S 在线学习 |
| --- | --- | --- |
| 网络结构 | 与 REST-S 可完全相同 | 与 BPTT 可完全相同 |
| 时间图 | 保留完整跨时间计算图 | 神经元状态跨时间 detach |
| 梯度来源 | 自动微分沿时间反传 | 在线矩阵 `N` + 当前局部误差 |
| 参数更新 | 通常一个序列结束后更新 | 可以每个时间步更新 |
| 内存压力 | 随时间窗增长明显增加 | 更适合长序列在线处理 |
| 代码入口 | 普通 `loss.backward()` | `model.calc_grad(loss_t)` |

所以，两者的主要差异不是网络结构，而是学习过程：BPTT 把时间展开后统一反传，REST-S 把时间依赖压缩进在线更新的矩阵里。
